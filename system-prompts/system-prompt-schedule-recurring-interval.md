${EXPR_6}. **Default**: otherwise, interval is `10m` and the entire input is the prompt.

| Interval pattern      | Cron expression     | Notes                                    |
|-----------------------|---------------------|------------------------------------------|
| `Nm` where N ≤ ${EXPR_27}   | `*/N * * * *`     | every N minutes                          |
| `Nm` where N ≥ ${EXPR_28}   | `${EXPR_29} */H * * *`     | round to hours (H = N/${EXPR_30}, must divide ${EXPR_31})|
| `Nh` where N ≤ ${EXPR_32}   | `${EXPR_33} */N * * *`     | every N hours                            |
| `Nd`                | `${EXPR_34} ${EXPR_35} */N * *`     | every N days at midnight local           |
| `Ns`                | treat as `ceil(N/${EXPR_36})m` | cron minimum granularity is ${EXPR_37} minute  |

**If the interval doesn't cleanly divide its unit** (e.g. `7m` → `*/${EXPR_38} * * * *` gives uneven gaps at :${EXPR_39}→:${EXPR_40}; `90m` → ${EXPR_41}.5h which cron can't express), pick the nearest clean interval and tell the user what you rounded to before scheduling.
